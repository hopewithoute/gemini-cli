## Summary

Adds support for a fully customizable `Status Line` in the Gemini CLI, allowing users to override the default footer using any executable shell command or script defined in `settings.json`.

Closes #<!-- insert issue number if there is one -->

## Details

- **Configuration:** Added `ui.statusLine.command` to `settings.schema.json`.
- **Execution:** Created `StatusLineService` that spawns a subprocess (e.g., `bash`, `node`, `jq`) with `AbortController` cancellation to prevent memory leaks and orphaned processes.
- **Data Injection:** The service injects a rich JSON payload representing the current session state via `stdin`. The payload includes:
  - Model info (`display_name`)
  - Workspace and Git branch
  - Deep telemetry (Context left %, total tokens, output tokens/sec).
- **UI Integration:** The `useCustomStatusLine` React hook polls the service whenever the state changes (debounced by 300ms) and dynamically renders the `stdout` (which supports ANSI colors) directly into `Footer.tsx`.
- **Documentation:** Added `docs/cli/custom-statusline.md` with setup guides and examples for using `jq` or an external `.sh` file.

## How to Validate

1. Enable the custom status line in `.gemini/settings.json`:
   ```json
   {
     "ui": {
       "statusLine": {
         "command": "jq -r '"\u001b[36m🤖 " + .model.display_name + "\u001b[0m | 🪙 Tokens: " + (.usage.prompt_tokens | tostring)'"
       }
     }
   }
   ```
2. Start the CLI (`npm run start`).
3. Observe the bottom of the screen; the default footer should be replaced with the custom formatted and colored text.
4. If you write a command that fails, verify the error text replaces the footer properly instead of crashing the Ink UI.
